# Балансировка нагрузки мессенджера MAX (Россия)

## 1. Функциональное разбиение по доменам

| Домен                | Функции                                   |
|---------------------|------------------------------------------|
| auth.max.ru          | Аутентификация, регистрация, сессии      |
| msg.max.ru           | Отправка/приём сообщений                  |
| media.max.ru         | Загрузка/скачивание медиа                |
| search.max.ru        | Поиск сообщений и чатов                   |
| profile.max.ru       | Работа с профилем пользователя           |

> Разделение доменов позволяет распределять нагрузку по типам запросов и оптимизировать инфраструктуру датацентров.

---

## 2. Расположение датацентров

| ДЦ           | Город           | Цель                                           |
|--------------|----------------|-----------------------------------------------|
| DC1          | Москва         | Основной рынок, высокая плотность пользователей |
| DC2          | Санкт-Петербург| Резервный регион, снижение задержки для СЗФО  |
| DC3          | Новосибирск    | Сибирь и Дальний Восток, локальная доступность |
| DC4          | Ростов-на-Дону | Южный регион, равномерное распределение нагрузки |

> Распределение по регионам снижает задержку и повышает стабильность сервиса.

---

## 3. Распределение запросов по типам и ДЦ

| ДЦ           | % пользователей | Средний RPS | Пиковый RPS |
|--------------|----------------:|------------:|------------:|
| Москва       | 40 %            | 28 800      | 72 000      |
| Санкт-Петербург | 25 %         | 18 000      | 45 000      |
| Новосибирск  | 20 %            | 14 400      | 36 000      |
| Ростов-на-Дону | 15 %           | 10 800      | 27 000      |

> Распределение учитывает плотность пользователей и типы запросов: send/receive messages, media upload/download, search.

---

## 4. Схема DNS балансировки

- Используется **GeoDNS** внутри России:
  - Запрос направляется на ближайший ДЦ по региону пользователя.
  - При недоступности региона — перенаправление на ближайший активный ДЦ.
